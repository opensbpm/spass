Class: TransitionCondition

Annotations (1)

  • rdfs:comment "An Transition condition defines an alternatives for a state that the transition the condition is attached to, originates from. An alternative (to leave the state) is only a real alternative if the condition is fulfilled (technically: if that according function returns "true")

    Note: Technically and during execution "conditions" belong to states. They define when it is allowed to leave that state. However, in PASS models exit conditions for states are defined and connected to the according transition edges. Therefore transition conditions are individual entities and not DataProperties.

    The according matching must be done by the model execution environment.

    By its existence, an edge/transition defines one possible follow up "state" for its state of origin. It is coupled with an "Exit Condition" for that state (the transition condition) that must be fulfilled in the originating state in order to leave the state."     @en
